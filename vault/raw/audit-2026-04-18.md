# Raw Audit — 2026-04-18

Source: [[iindev.audit.2026-04-18]] (Obsidian)

## Scope
Полный аудит frontend/services/ayan + app/composables + app/types + backend/routes/api.php

## Current State
- Frontend: 7 страниц ayan с mock-API (все используют useTaxiAPI → useMockAPI)
- Backend: 0 контроллеров, 0 моделей, 0 миграций. Только роуты в api.php
- Auth: сломана (useAuth не вызывается, токен не персистится, в мок-режиме фейлится)

## Critical Bugs (P0 — сломается при переходе на реальный API)
1. Статусы не совпадают: мок searching/matched/arrived/on-trip vs реальный API open/accepted
2. Поля не совпадают: мок pickup/destination/passengerId vs реальный from_address/to_address/passenger_id
3. Driver объект: мок {name, rating} vs реальный User {first_name, username}
4. User тип: мок добавляет isAvailable/stats, реальный User этого не имеет

## Bugs (P1 — прямо сейчас в мок-режиме)
1. acceptOrder не ставит myActiveOrder — белый экран на active-ride
2. my-order.vue — пустой экран когда нет активного заказа
3. onBackButtonClicked накапливает обработчики — нет offClick при unmount
4. complete.vue рейтинг не работает — 5 звёзд без обработчика
5. useAuth.login глотает ошибки — нет фидбека пользователю

## UX Issues (P2)
1. Двойная кнопка назад (layout header + Telegram BackButton)
2. Нет поллинга на active-ride/orders — изменения не видны
3. USwitch мерцает при ошибке (v-model переключает до API ответа)
4. 20+ захардкоженных русских строк без i18n
5. Валидация цены только > 0, нет min/max

## Minor (P3)
1. variant="secondary" в my-order.vue — не существует в Nuxt UI v4
2. MOCK_CONFIG.autoAcceptDelay не используется (hardcoded 5000)
3. Мок route matching хрупкий
4. useAPI.ts — useState внутри computed, Content-Type на GET запросах
5. useTaxiAPI.ts — оба API инициализируются всегда

## Frontend-Backend Endpoint Mismatches

| POST /ayan/orders/{id}/arrive | Не существует | MISS |

## Architectural Issues
1. Нет общих компонентов — весь UI инлайн в страницах
2. statusConfig дублируется в my-order.vue и active-ride.vue
3. formatPrice() дублируется в orders.vue и active-ride.vue
4. Нет контекста заказа на complete.vue
5. Рейтинг — мёртвая фича (UI есть, API нет)
6. Токен не персистится (useState сбрасывается при refresh)
7. Нет авто-логина
8. Поллинг хаотичный (5-10 сек, нет бэкоффа)
9. TaxiOrder.driver — double-optional (? + | null)
10. useTg().ready() никогда не вызывается
