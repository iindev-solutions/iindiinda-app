# AYAN (Бардыбыт) — Vision & Concept

> AYAN — это место, где: люди, которые едут → находят тех, кому по пути

**Суть**: доска попуток, где люди публикуют поездки или запросы, ищут попутчиков и договариваются напрямую. Без посредников. Без контроля. Без оплаты внутри платформы.

---

## Основная идея

Каждый день люди ездят: на работу, между улусами, по делам. Часто одни, хотя в машине есть свободные места.

В то же время другие ищут, как доехать.

AYAN объединяет этих людей.

---

## Как это работает

Всё просто:
1. Человек создаёт **поездку** или **запрос**
2. Другие это видят
3. Откликаются
4. Договариваются напрямую

Без посредников. Без сложных систем.

---

## Роль AYAN

AYAN **НЕ**:
- управляет поездками
- контролирует процесс
- является такси

AYAN **НЕ является** мгновенным вызовом, алгоритмами или картами.

AYAN:
→ помогает людям найти друг друга

---

## Сценарий 1 — Регулярные поездки

Человек ездит каждый день по одному маршруту.

**Пример:** ДСК → Порт каждое утро в 8:30.

Создаёт поездку:
> "Езжу каждый день ДСК → Порт, выезд в 8:30, есть 2–3 места"

Другие люди (живущие рядом, едущие туда же) находят и откликаются.

**Суть:** Это не разовая поездка. Это формирование постоянных связей и маршрутов.

---

## Сценарий 2 — Разовые поездки

Человек едет один раз.

**Пример:** Якутск → Намцы сегодня в 18:00.

Создаёт поездку:
> "Еду Якутск → Намцы в 18:00, есть 2 места"

Другие находят, откликаются, договариваются.

---

## Сценарий 2b — Обратный вариант (запрос)

Пассажир создаёт запрос:
> "Нужно доехать до Хандыги завтра"

Водители видят запрос, откликаются, предлагают поездку.

**Суть:** Быстрый способ доехать "здесь и сейчас".

---

## MVP (фокус)

Минимум на старте:

1. **Создать поездку** (водитель: откуда, куда, когда, места, цена)
2. **Создать запрос** (пассажир: откуда, куда, когда)
3. **Список поездок/запросов**
4. **Откликнуться**

ВСЁ.

---

## Главное правило

Не усложнять.

Идеи → карты, авто-подбор, трекинг — это НЕ AYAN на старте.

## Структура данных (MVP)

### Поездка (Ride)
```typescript
interface Ride {
  id: number
  from: string        // "Якутск"
  to: string          // "Намцы"
  date: string        // "2026-04-20"
  time: string        // "18:00"
  seats: number       // 2
  price: number | null // 500 или null (договорная)
  comment?: string
  driver: User
  status: 'open' | 'closed'
  created_at: string
}
```

### Запрос (Request)
```typescript
interface Request {
  id: number
  from: string        // "Хандыга"
  to: string          // "Якутск"
  date: string        // "2026-04-21"
  comment?: string
  passenger: User
  status: 'open' | 'fulfilled'
  created_at: string
}
```

### Пользователь (User)
```typescript
interface User {
  id: number
  first_name: string
  username?: string
  phone?: string
}
```

### Отклик (Response)
```typescript
interface Response {
  id: number
  ride_id?: number    // один из
  request_id?: number // или другой
  user: User
  message: string
  created_at: string
}
```

---

## Status Flow

Только два статуса:

- **Поездка:** `open` → `closed` (места закончились)
- **Запрос:** `open` → `fulfilled` (нашёлся попутчик)

Нет промежуточных статусов. AYAN не следит за процессом.

---

## API Endpoints (MVP)

```
# Поездки
GET    /ayan/rides              — список открытых поездок
POST   /ayan/rides              — создать поездку
GET    /ayan/rides/:id          — детали поездки
PATCH  /ayan/rides/:id          — закрыть поездку (owner only)

# Запросы
GET    /ayan/requests           — список открытых запросов
POST   /ayan/requests           — создать запрос
GET    /ayan/requests/:id       — детали запроса

# Отклики
POST   /ayan/rides/:id/respond  — откликнуться на поездку
POST   /ayan/requests/:id/respond — откликнуться на запрос

# Контакты
GET    /ayan/rides/:id/contact  — получить контакт водителя (после отклика)
GET    /ayan/requests/:id/contact — получить контакт пассажира (после отклика)
```

## Architecture Notes

```
frontend/services/ayan/app/
├── pages/
│   ├── ayan.vue                → /ayan (parent, <NuxtPage />)
│   └── ayan/
│       ├── index.vue           → /ayan (hub: rides list + my)
│       ├── create.vue          → /ayan/create (create ride)
│       ├── request.vue         → /ayan/request (create request)
│       ├── ride/:id.vue        → /ayan/ride/:id (ride details + respond)
│       └── my-rides.vue        → /ayan/my-rides (my rides/requests)
```

**Service layer:** `frontend/services/ayan/` — Nuxt layer, extends base app.

**No complex state.** Простой fetch → render. Поллинг не нужен ( данные не меняются часто).

**No ITaxiAPI.**直接 useAPI() — убираем лишний слой абстракции.

---

## Что НЕ делаем на старте

- Карты и геокодинг
- Трекинг в реальном времени
- Автоматический подбор
- Внутриплатёжный расчёт
- Рейтинги и отзывы
- push-уведомления

Всё это — после MVP, если будет спрос.
