# iind.app — Vision & Concept

> iindiinda — делаем сложные вещи просто. Modern solutions. Simply.
> **Сделано сахалар для сахалар.**

**Суть**: платформа из 4 сервисов, где люди решают повседневные задачи через других людей. Без посредников. Без сложных систем. Просто, понятно, по-нашему.

---

## Продуктовые принципы (основа всех решений)

Если новая идея противоречит хотя бы одному — она не реализуется.

### 1. Простота или смерть
Любая функция должна быть понятна за 3 секунды.
Если пользователь должен думать — функция слишком сложная.

### 2. Поток прежде монетизации
Нет 100 заявок/день — нет оплаты.
Сначала активность, потом деньги.

### 3. Люди договариваются сами
Платформа только соединяет. Не контролирует. Не гарантирует.
Всё остальное — напрямую между людьми.

### 4. Один паттерн для всех
Все 4 сервиса работают по одной логике:
→ заявка / доступность / отклик
Новая логика = ошибка архитектуры.

### 5. Своё > чужое
Не копируем Яндекс, Авито, сложные маркетплейсы.
Делаем как у нас — просто, по-нашему, под наши реальности.

---

## Почему именно для саха народа

Мы живем в регионе, где:
- Расстояния огромные (Якутск — Намцы — Хандыга)
- Люди регулярно перемещаются между улусами и городами
- Знакомые помогают знакомым: "кто едет в Намцы?", "кто знает мастера?"
- Нет удобных локальных сервисов — только общие федеральные

**iind.app берёт эту культуру взаимопомощи и делает её технологичной.**

Не привозим решения извне.
Делаем **свои**, **понятные**, **простые**.

## Что такое iind.app

Это не "4 отдельных сервиса".

Это:
→ **единый слой взаимодействия между людьми**

Один принцип для всего:
> Человек создаёт запрос или показывает доступность → другие откликаются → договариваются напрямую

## 4 сервиса — 4 повседневных задачи

Мы взяли задачи, которые саха люди решают каждый день:

| Сервис          | Задача                | Пример                    |
| --------------- | --------------------- | ------------------------- |
| **AYAN** (Айан) | Передвижение          | "Еду в Намцы, есть места" |
| **UUS** (Уус)   | Решение бытовых задач | "Нужно помыть окна"       |
| **TAL** (Тал)   | Запись к мастеру      | "Свободен ли парикмахер?" |
| **AGAL** (Аҕал) | Отправка вещей        | "Кто летит в Москву?"     |

**Вместе они покрывают 80% повседневных нужд.**

---

## Единый принцип — интуитивно для всех

Выучил один раз — пользуйся всеми:

```
Создать поездку (AYAN)
    =
Создать задачу (UUS)
    =
Показать доступность (TAL)
    =
Создать маршрут (AGAL)
```

**Один интерфейсный паттерн** — нет барьера входа.

## Национальный контекст

### Для кого это:
- Студенты, едущие домой в улус
- Мастера, ищущие клиентов
- Мамы, которым нужна помощь по дому
- Предприниматели, отправляющие документы
- Водители, которым скучно ехать одним

### Почему это работает:
- **Простой язык** — без англицизмов там, где не нужны
- **Привычные паттерны** — "кто едет?", "кто знает?"
- **Быстро** — решение за минуты, не часы
- **Бесплатно для заказчиков** — платят только исполнители

---

## Платформенная ценность (не просто сумма сервисов)

### 1. Единый профиль
Один аккаунт — все 4 сервиса.
Зарегистрировался раз — пользуйся всем.

### 2. Кросс-активность
- Водитель AYAN видит запросы UUS по маршруту
- Мастер TAL может откликаться на UUS-задачи
- Пассажир AGAL может стать водителем AYAN

### 3. Единое доверие
Репутация в одном сервисе = доверие в другом.
Хороший мастер в TAL = надёжный исполнитель в UUS.

### 4. Локальный рынок
Создаём **внутренний рынок услуг и взаимодействий** внутри региона.
Деньги остаются внутри сообщества.

---

## Монетизация платформы

**Принцип: бесплатно для заказчиков, платит исполнитель.**

Это честно:
- Заказчик рискует (не знает результат)
- Исполнитель получает доход (он зарабатывает)
- Платформа соединяет (и берёт комиссию/подписку)

### Модели по сервисам:

| Сервис | Модель | Кто платит |
|--------|--------|------------|
| **AYAN** | Продвижение + подписки | Водитель |
| **UUS** | Отклики + подписка | Исполнитель |
| **TAL** | Подписка + продвижение | Мастер |
| **AGAL** | Комиссия + страховка | Перевозчик + отправитель |

### Платформенная монетизация:
- **Кросс-продвижение** — "мастер TAL видит запросы UUS"
- **Пакетные подписки** — "доступ ко всем сервисам 500 ₽/мес"
- **Аналитика** — анонимизированные инсайты о рынке

- 1 000 транзакций/день
- Все крупные населённые пункты Якутии

- 50 000+ пользователей
- 3 000+ транзакций/день
- Расширение на соседние регионы
- Полноценная экосистема
