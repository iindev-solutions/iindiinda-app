	# TAL (Тал) — Vision & Concept

> TAL — это простой способ записаться к мастеру. Без сложных календарей. Без переписок. Без лишнего.

**Суть**: мастера показывают свою доступность, клиенты видят кто свободен и сразу записываются. Без длительных договорённостей. Без ожиданий ответа.

---

## Основная идея

Когда человеку нужна услуга:
- стрижка
- маникюр
- массаж
- любая бытовая услуга

он не хочет:
- искать аккаунты в Instagram
- писать в личку и ждать ответа
- договариваться о времени

Он хочет:
→ быстро понять, кто свободен
→ и сразу записаться

---

## Как это работает

В центре TAL — не сложное расписание, а **простое состояние мастера**.

Мастер может за секунду показать:
- 🟢 **Свободен сейчас** — можно записаться прямо сейчас
- 🕓 **Свободен позже** — сегодня, но позже
- 📅 **Свободен завтра** — запись на будущее
- 🔴 **Занят** — временно не принимает

Клиент:
1. Заходит в TAL
2. Видит список мастеров с их статусами
3. Выбирает подходящего
4. Нажимает "Записаться" или "Написать"
5. Контакт открывается, договариваются напрямую

**Всё.**

---

## Роль TAL

TAL **НЕ**:
- управляет расписанием мастера
- контролирует записи
- берёт комиссию с услуг
- заменяет CRM мастера

TAL:
→ показывает, кто свободен прямо сейчас
→ устраняет барьер поиска и договорённости

---

## Сценарий 1 — Запись "сейчас" (ядро)

**Классическая ситуация:**
Клиенту нужна стрижка сегодня, и он хочет понять, кто свободен.

**Как работает:**
1. Мастер нажимает "🟢 Свободен сейчас"
2. Появляется в списке с бейджем
3. Клиент видит, нажимает "Записаться"
4. Мастер получает уведомление (или просто видит в списке записавшихся)
5. Клиент приходит
6. Мастер ставит себе "🔴 Занят" на время работы
7. После — снова "🟢 Свободен"

**Суть:** Мгновенная запись без переписки.

---

## Сценарий 2 — Запись на конкретное время

Клиент хочет записаться вечером, а не сейчас.

**Как работает:**
1. Мастер ставит "🕓 Свободен сегодня" и может указать примерное время ("после 18:00")
2. Клиент видит, нажимает "Записаться"
3. В сообщении клиент пишет желаемое время
4. Договариваются

**Суть:** Гибкость без сложного календаря.

---

## Сценарий 3 — Запрос (fallback)

Клиент не нашёл подходящего мастера или никто не свободен.

**Как работает:**
1. Клиент создаёт запрос: "Хочу на стрижку сегодня вечером"
2. Указывает категорию (стрижка, маникюр, etc.)
3. Мастера этой категории получают уведомление / видят в ленте запросов
4. Свободные мастера откликаются
5. Клиент выбирает из откликов

**Суть:** Когда нет готового предложения — создаём спрос.

---

## Сценарий 4 — Постоянный мастер

Клиент нашёл мастера, которому доверяет.

**Как работает:**
1. После первой записи клиент добавляет мастера в "Избранное"
2. Видит только его статус
3. Быстро записывается повторно
4. Формируется долгосрочная связь

**Суть:** Удобство для постоянных клиентов.

---

## Типы услуг — 3 категории

Чтобы упростить поиск:

| Категория | Примеры |
|-----------|---------|
| 💇 **Красота** | стрижка, маникюр, педикюр, брови, ресницы |
| 🏠 **Дом** | массаж, уборка, помощь |
| 🔧 **Ремонт** | бытовые услуги (маленькие задачи) |

*На MVP: можно начать с одной категории (красота)*

---

## Статусы мастера

Простая система без сложных календарей:

| Статус | Значение | Для клиента |
|--------|----------|-------------|
| 🟢 **Свободен сейчас** | Готов принять клиента | Запишись прямо сейчас |
| 🕓 **Свободен позже** | Сегодня, но позже | Напиши, договорись |
| 📅 **Свободен завтра** | Запись на будущее | Запланируй визит |
| 🔴 **Занят** | В работе или не принимает | Подожди или найди другого |

Мастер меняет статус в **один тап**.

---

## Механика записи

**Вариант A — Простой:**
Клиент нажимает "Записаться" → Контакт открывается → Договариваются напрямую

**Вариант B — С указанием времени:**
Клиент нажимает "Записаться" → Пишет желаемое время → Контакт открывается → Уточняют

**Вариант C — Через запрос:**
Клиент создаёт запрос → Мастера откликаются → Клиент выбирает → Контакт открывается

**Важно:** TAL не бронирует время. Только показывает доступность и соединяет.

---

## MVP (фокус)

Минимум на старте:

1. **Мастер меняет статус** (4 состояния)
2. **Клиент видит список мастеров** (фильтр по категории)
3. **Записаться** (открыть контакт)
4. **Запрос** (если никто не свободен)

ВСЁ.

---

## Что НЕ делаем на старте

- ❌ Сложные календари и слоты времени
- ❌ Бронирование времени (платформа не блокирует)
- ❌ CRM для мастеров
- ❌ История записей (сложная)
- ❌ Рейтинги и отзывы
- ❌ Онлайн-оплата
- ❌ Push-уведомления (пока)
- ❌ Чат внутри приложения
- ❌ Подписки / премиум

**Главное правило:** Не превращать в "большой сервис". Минимум для быстрой записи.

# 💰 Потенциал и монетизация

## Во что это может вырасти

TAL может вырасти в:

→ основной способ записи к мастерам в регионе
→ ежедневный инструмент для мастеров
→ замена Instagram/WhatsApp переписок

---

## Поведение рынка

- у мастеров есть пустые окна в расписании
- клиенты ищут "прямо сейчас"
- запись происходит каждый день
- сейчас всё через лички: пишут, ждут ответа, теряют время

Это:
→ постоянный поток записей
→ регулярность использования
→ ежедневный трафик

---

## Роль в экосистеме iind

TAL даёт:

→ **регулярность** — мастера и клиенты возвращаются ежедневно
→ **трафик** — постоянный поток пользователей
→ **возвращаемость** — привычка использовать TAL

TAL — это **retention-драйвер** экосистемы. Люди возвращаются регулярно.

---

## Монетизация (после роста)

### 1. Подписка мастеров

Мастер платит за:
- доступ к клиентам
- размещение в списке
- приоритет в поиске

→ стабильный recurring доход

### 2. Продвижение

Мастер платит за:
- выделение в списке "🟢 свободен сейчас"
- топ-позицию
- бейдж "рекомендован"

→ performance-маркетинг

### 3. Комиссия (позже)

Процент с записи.

→ требует масштаба и доверия

---

## Почему будут платить

Потому что:
→ TAL даёт клиентов напрямую
→ экономит время на переписках
→ заполняет пустые окна в расписании

**ROI для мастера:** 1 новый клиент = окупаемость подписки.

---

## Потенциал дохода (оценка)

**Если:**
- 50 активных мастеров
- 200–500 записей в день
- подписка 300 ₽/мес

→ 50 × 300 ₽ = 15 000 ₽/мес (база)
→ + продвижение = 30 000–50 000 ₽/мес

**При росте:**
- 200 мастеров → 60 000 ₽/мес (подписки)
- + продвижение + комиссия = масштаб

---

## Долгосрочное развитие

**После стабильного потока:**
- лёгкий календарь (простые слоты)
- история записей (для клиентов)
- базовый профиль мастера (фото, работы)
- рейтинг (доверие)

**Но только после роста.**

---

## Главное правило

Не усложнять. Не превращать в "большой сервис".

Сначала:
→ быстрые записи
→ доступность
→ поток пользователей

Потом:
→ деньги

## Decision Log

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| Тип сервиса | Доступность + запись | Не CRM, не календарь |
| Контроль | Нет | TAL не управляет расписанием |
| Оплата | Напрямую клиент → мастер | Платформа не участвует |
| Статусы | 4 простых | Один тап для мастера |
| Бронирование | Нет | Только показываем доступность |
| Запросы | Да (fallback) | Когда нет готового предложения |
| Роль в экосистеме | Retention-драйвер | Регулярный трафик |
| Монетизация | Не на MVP | Сначала поток, потом деньги |
